# eikosany

> Algorithmic Composition tools for Erv Wilson's Combination Product Sets (CPS).

## Overview

`eikosany` is a specialized R package and accompanying interactive application (`xen_assist`) designed to compute microtonal scales derived from harmonic product sets. Its primary purpose is to translate the complex mathematical theories of Erv Wilson—specifically the 20-note Eikosany and other Combination Product Sets—into practical data structures that can be used for music composition and synthesizer tuning.

The project bridges the gap between music theory (just intonation ratios) and production (MIDI notes and cents offsets), providing developers and composers with a way to generate, visualize, and hear these non-standard tunings.

## Design Philosophy & Decisions

The project is built on a **Backend-First** architecture where all theoretical calculations are decoupled from the presentation layer. This allows the `eikosany` package to function as a standalone library for researchers while powering the interactive `xen_assist` dashboard.

### Key Architectural Decisions
*   **Decoupled Backend:** Core logic in `R/cps_functions.R` is agnostic of the UI, accepting parameters (harmonics, divisors) and returning standardized data objects.
*   **Data Table Core:** Every major output (scale tables, keyboard maps, interval lists) uses `data.table` for high-performance sorting and manipulation.
*   **Lean Frontend:** To maximize responsiveness in the Shiny app, visualization favors base R/grid graphics over heavy frameworks like `ggplot2`.
*   **Multimodal Interface:** Interaction occurs through a combination of traditional UI elements (tables, plots) and an AI-driven `btw_app` chat window for natural language control.
*   **Direct Audio Feedback:** Integration with `tuneR` and `seewave` allows the system to synthesize WAV files from calculated ratios, providing immediate sonic validation of theoretical scales.

## How It Works

The project follows a linear data pipeline from theoretical definition to synthesizer implementation:

### 1. Harmonic Definition $\rightarrow$ Ratios
Users define a set of harmonic factors (e.g., the first six odd numbers)."Combination Product Sets are generated by taking combinations of these factors, multiplying them, and reducing them into a specific period (usually an octave) using `period_reduce`.

### 2. Ratios $\rightarrow$ Scale Tables
Using functions like `cps_scale_table` or `ps_scale_table`, the ratios are converted into a **Scale Table**. This table includes:
*   **Note Name:** The product label (e.g., "1x3x5").
*   **Ratio:** The numerical ratio and its vulgar fraction representation via the `fractional` package.
*   **Cents:** The distance from the root in cents ($1200 \cdot \log_2(\text{ratio})$).

### 3. Scale Tables $\rightarrow$ Implementation Maps
To actually play these notes, the software converts scale tables into:
*   **Offset Matrices (`offset_matrix`):** Cents offsets applied to standard 12-TET keys.
*   **Keyboard Maps (`keyboard_map`):** A full MIDI mapping (note number $\rightarrow$ frequency) for synthesizers with arbitrary tuning capabilities.

## The `xen_assist` Application

The `xen_assist` Shiny dashboard provides an interactive window into this pipeline. Its primary components include:

*   **AI Chat Interface:** A `btw_app` window that allows users to manipulate scales and request configurations via natural language.
*   **Interactive Data Viewer:** A read-only, RStudio-style table display for exploring scale tables, interval lists, and keyboard maps.
*   **Landscape Plot Window:** Simple base/grid graphics visualization of the scale's structure.
*   **WAV Player:** A playback system that synthesizes sine waves based on calculated frequencies to let users hear the results.

### Key Components

| Component | Location | Purpose |
| :--- | :--- | :--- |
| `cps_scale_table()` | `R/cps_functions.R` | Generates a scale table from harmonic factors and a root divisor. |
| `keyboard_map()` | `R/cps_functions.R` | Maps a microtonal scale across the full MIDI range (0-127). |
| `offset_matrix()` | `R/cps_functions.R` | Calculates cents offsets relative to 12-TET keys. |
| `cps_chord_table()` | `R/cps_functions.R` | Computes harmonic and sub-harmonic chord pairs using Wilson's notation. |
| `period_reduce()` | `R/cps_functions.R` | Mathematically wraps a ratio into the [1, period) range. |

## Developer Orientation

### Getting Started
New developers should start by reading `vignettes/introduction-to-combination-product-sets.Rmd` to understand the music theory foundations before diving into the code.

### Implementation Roadmap
When extending the project:
1.  **Backend Logic:** Add any new mathematical transformations or ratio types to `R/cps_functions.R`. Ensure they return a `data.table` compatible with existing map functions.
2.  **Shiny Integration:** Connect backend functions to the `xen_assist` UI. Use the AI chat window as an orchestration layer that calls these functions based on user intent.
3.  **Audio Logic:** When implementing new playback features, use `tuneR` for wave generation and serve them as temporary WAV files in the Shiny session.

### Known Complexities
The combinatorial logic in `cps_chord_table` is the most mathematically dense part of the codebase. It implements Erv Wilson's "n choose m" notation—carefully review the internal comments if modifying this section.
